GitHub

GitHub - это сервис для совместной разработки и хостинга проектов.

С помощью GitHub над кодом проекта может работать неограниченное количество программистов из любых точек мира.

В GitHub есть система контроля версий Git: сервис позволяет просматривать и контролировать любые изменения кода любым разработчиком и возвращаться к состоянию до изменений.

История создания GitHub

GitHub был создан в 2008 году Томом Престон-Вернером и его командой разработчиков.

Они столкнулись с необходимостью более эффективного сотрудничества над проектами с открытым исходным кодом. Команда решила создать платформу, которая бы предоставляла простой и удобный интерфейс для хранения и совместной работы над проектами. Так появился GitHub.

Git Bush

Git Bash — это приложение для Windows с интерфейсом командной строки (CLI), которое позволяет взаимодействовать с системой контроля версий Git.

С помощью Git Bash можно:

  1. клонировать репозитории;

  2. фиксировать изменения;

  3. отправлять и получать изменения;

  4. автоматизировать ручные задачи с помощью написанных пользователем скриптов.

Основные команды Git:

  • git status

    проверка статуса репозитория

  • git commit

    добавление файлов в репозиторий

  • git add

    добавление файлов в индекс

  • git push

    запушить изменения

GitLab

GitLab — это онлайн-сервис, предназначенный для работы с git-репозиториями.

Его можно использовать непосредственно на официальном сайте (gitlab.com), зарегистрировав аккаунт, или установить и развернуть на своём сервере.

Отличия GitHub от GitLab

  1. GitHub — это облачный сервис, принадлежащий Microsoft, на котором можно хранить свои проекты и управлять ими. В отличие от него, GitLab можно развернуть на собственном сервере.

  2. В GitHub для работы с CI/CD придётся использовать дополнительный инструмент — GitHub Actions. GitLab — это полностью интегрированное решение со встроенными инструментами CI/CD.

  3. В бесплатной версии GitHub в приватных репозиториях используется два уровня доступа: владелец (owner) и соавтор (collaborator). В GitLab таких ограничений нет. Количество пользователей с разным уровнем доступа не ограничено.

Чтобы загрузить проект на GitHub, выполните следующие шаги:

  1. Создайте репозиторий на GitHub

  2. Клонируйте репозиторий на локальный компьютер

  3. Добавьте проект в локальный репозиторий

  4. Загрузите проект на GitHub
